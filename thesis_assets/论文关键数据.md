## 二、模型性能对比

### 2.1 主要模型结果

| 模型                    | Test Macro F1 | Test Accuracy | 备注           |
| ----------------------- | ------------- | ------------- | -------------- |
| **BERT 15分钟窗口**     | **0.3770**    | **0.3819**    | **最佳模型**   |
| BERT 30分钟窗口         | 0.3332        | 0.3668        | 窗口过长，标注噪声大 |
| BERT HC过滤             | 0.3483        | 0.3579        | 过度过滤，泛化下降 |
| 基线模型 (TF-IDF + SVM) | 0.3359        | 0.3390        | 同一测试集(3,823条) |
| BERT 无输入增强          | 0.3342        | 0.3450        | 消融对比       |

**关键发现**：
- BERT 15分钟窗口+增强模型相比基线 Macro F1 提升 **+12.2%**
- 输入增强策略提升 **+12.8%**（从0.3342到0.3770）
- 30分钟窗口性能下降，说明15分钟是更优的事件窗口
- 高置信度过滤减少45%训练数据，反而降低了泛化能力

### 2.2 各类别性能详情（最佳模型：BERT 15分钟窗口）

| 类别           | Precision | Recall | F1-Score | Support |
| -------------- | --------- | ------ | -------- | ------- |
| 看跌 (Bearish) | 0.3901    | 0.2698 | 0.3190   | 1,290   |
| 中性 (Neutral) | 0.3433    | 0.4680 | 0.3961   | 1,032   |
| 看涨 (Bullish) | 0.4127    | 0.4191 | 0.4159   | 1,501   |

**分析**：
- 看涨类别表现最好（F1 = 0.4159）
- 看跌类别Recall最低（0.2698），大量利空样本被误判
- 中性类别Recall最高（0.4680）但Precision较低

## 三、RAG检索增强系统

### 3.1 RAG配置
| 参数       | 值         |
| ---------- | ---------- |
| 文档块数量 | 633 chunks（清洗后544） |
| 向量模型   | BAAI/bge-m3（1024维）   |
| 检索Top-K  | 3          |
| 向量数据库 | ChromaDB   |

### 3.2 RAG检索策略对比实验结果

| 配置                    | 文档块数 | 平均相关性 | 相关性变化 | 平均延迟 | 延迟变化   |
| ----------------------- | -------- | ---------- | ---------- | -------- | ---------- |
| Baseline（原始）        | 573      | 0.5777     | -          | 6.71ms   | -          |
| Exp A（清洗）           | 532      | 0.5770     | -0.1%      | 5.59ms   | -16.7%     |
| Exp B（清洗+元数据）   | 532      | 0.5624     | -2.6%      | 4.60ms   | **-31.4%** |

**关键发现**：
- 数据清洗去除7.2%噪声文档块，延迟降低16.7%
- 元数据过滤使延迟降低31.4%，但相关性下降2.6%（因缩小搜索范围）
- 相关性下降完全来自有过滤的查询，无过滤查询三组表现一致
- 元数据过滤的实际价值在于结果可控性、语言一致性和时效性
